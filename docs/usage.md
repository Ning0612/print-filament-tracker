## 首頁（Dashboard）

`GET /`

顯示系統概覽：

- **統計卡片**：總任務數、已對應耗材數、活躍耗材捲數、總列印重量
- **耗材經濟面板**：
  - 重量統計：總初始重量、已用量（進度條 + 百分比）、剩餘量
  - 成本統計：耗材總購入成本、已消耗成本與剩餘成本（需在 Spool 設定價格）
- **最近任務**：最近 10 筆列印記錄（任務名稱、日期、時長、重量、列印狀態）

點擊任務名稱可進入任務詳情頁。

## 列印歷史

### 任務列表

`GET /tasks/`

- **搜尋**：輸入任務名稱或列印機名稱篩選
- **分頁**：預設每頁 20 筆，可調整頁數
- **欄位**：任務名稱、封面圖縮圖、列印機、開始時間、時長、重量、耗材數量、**列印狀態徽章**

**列印狀態徽章**：雲端同步任務會依 API 回傳的 status 顯示：
- `completed`（綠色）
- `failed`（紅色）
- `in progress`（灰色，中途同步的任務）

手動新增的任務不顯示狀態徽章。

### 任務詳情

`GET /tasks/<id>`

- 封面圖（點擊放大至全螢幕 Lightbox）
- 基本資訊：時間、時長、列印機、總重量、列印狀態
- **板片資訊**：板片編號（Plate Index）與板片名稱（若 API 有提供）
- **耗材使用明細**：每個耗材槽的材料、顏色、用量、對應的 Spool
  - 點擊 Spool 名稱可跳轉至 Spool 詳情頁
  - 點擊「重新對應」可直接在此頁更換 Spool，無需前往耗材對應頁
- 手動任務可在此頁進行編輯或刪除

### 新增手動任務

`GET/POST /tasks/new`

雙欄佈局，左側封面圖、右側表單：

| 欄位 | 說明 |
|------|------|
| 任務名稱 | 必填 |
| 列印機 | 選擇已建立的列印機 |
| 開始時間 | 格式：`YYYY-MM-DD HH:MM:SS` |
| 結束時間 | 格式同上；輸入後自動計算耗時 |
| 耗時（秒） | 可手動輸入，或由時間欄位自動計算 |
| 封面圖 | 上傳 PNG / JPG / WEBP / GIF，最大 10 MB |
| 耗材行 | 可新增多行：材料、顏色（#RRGGBB）、重量（克） |

點擊「加入耗材行」可新增多筆；重量欄位自動加總至「總重量」。

### 編輯手動任務

`GET/POST /tasks/<id>/edit`

僅 `is_manual = 1` 的任務可編輯，表單與新增相同。

### 刪除手動任務

`POST /tasks/<id>/delete`

刪除後一併清除封面圖檔案。僅手動任務可刪除。

## 耗材管理（Spools）

### Spool 列表

`GET /spools/`

耗材依狀態分群顯示：

| 狀態 | 說明 |
|------|------|
| **active** | 使用中，剩餘 ≥ 10% |
| **low** | 剩餘 < 10% |
| **sealed** | 未開封（未設定 `opened_at`）|
| **empty** | 剩餘量 ≤ 0 |

每行顯示：顏色色票（含十六進位色碼與 RGB 值）、材料、初始重量、已使用量（進度條）、剩餘量、狀態徽章。

### Spool 詳情

`GET /spools/<id>`

顯示單捲耗材的完整資訊：

- **基本資料**：材料、顏色（色票 + Hex + RGB）、初始重量、價格、購買/開封日期
- **使用統計**：已用重量、剩餘重量、使用率進度條、狀態
- **列印歷史**：使用此耗材的所有列印任務，附任務名稱、日期、用量

### 新增 Spool

`GET/POST /spools/new`

| 欄位 | 說明 |
|------|------|
| UID | 唯一識別碼（建議使用廠商 UID 或自訂字串） |
| 材料 | 如 PLA, PETG, ABS, TPU |
| 顏色名稱 | 自訂顏色名稱 |
| 顏色（Hex） | `#RRGGBB` 格式 |
| 初始重量（克） | 必填 |
| 價格 | 可選，用於成本分析 |
| 購買日期 | 可選 |
| 開封日期 | 設定後狀態從 sealed → active |
| 產品網址 | 可選 |
| 備注 | 可選 |

### 編輯 / 刪除 Spool

`GET/POST /spools/<id>/edit`、`POST /spools/<id>/delete`

> 注意：刪除 Spool 前，確認無對應的列印任務耗材；若有，對應關係將被清除。

### 匯入 Spool

`POST /spools/import`

支援 JSON 或 CSV 格式。

**JSON 格式：**
```json
{
  "version": "1.0",
  "exported_at": "2024-04-27T15:30:00",
  "spools": [
    {
      "uid": "SPOOL001",
      "material": "PLA",
      "color_name": "橘紅",
      "color_hex": "#FF6B35",
      "initial_weight_g": 1000.0,
      "price": 350.0
    }
  ]
}
```

**CSV 格式：**
```
uid,material,color_name,color_hex,initial_weight_g,price,purchased_at,opened_at
SPOOL001,PLA,橘紅,#FF6B35,1000.0,350.0,2024-01-01,2024-01-05
```

重複的 UID 自動跳過。

### 匯出 Spool

- `GET /spools/export/json`：下載 JSON 格式，包含所有 Spool 資料與對應關係
- `GET /spools/export/csv`：下載 CSV 格式（扁平化）

## 列印機管理（Printers）

### Printer 列表

`GET /printers/`

卡片形式顯示各列印機：機器圖片、名稱、型號、設備 ID、使用統計（任務數、總重量、總時長）。

### 新增 / 編輯 Printer

`GET/POST /printers/new`、`GET/POST /printers/<id>/edit`

| 欄位 | 說明 |
|------|------|
| 名稱 | 必填，唯一 |
| 設備 ID | Bambu 設備識別碼（如 `00M03XXXXXX`） |
| 型號 | 如 X1 Carbon、P1S |
| 購買日期 | 可選 |
| 圖片 | 上傳機器照片，最大 10 MB |
| 備注 | 可選 |

> 設備 ID 需與 Bambu Cloud API 回應中的 `deviceId` 一致，才能自動關聯列印任務。

## 耗材對應（Mapping）

`GET /mapping/`

三個頁籤：

### 未對應（Unmapped）

列出所有尚未關聯 Spool 的耗材記錄（`filament_spool_id IS NULL`，且未標記忽略）。

每行顯示：**列印狀態徽章**、任務名稱、開始時間、耗材槽、材料、顏色、用量。

**操作：**
- 下拉選單選擇 Spool → 點擊「對應」：建立關聯
- 點擊「忽略」：標記此耗材無實體對應（如廢料、樣品）

對應後該行立即以 HTMX 更新，無需整頁重載。

### 已對應（Mapped）

已成功關聯 Spool 的耗材記錄。每行亦顯示列印狀態徽章。

**操作：**
- 點擊「解除對應」：移除關聯（恢復為未對應狀態）
- 點擊「重新對應」：更換為不同 Spool

### 已忽略（Ignored）

標記為忽略的耗材記錄。每行亦顯示列印狀態徽章。

**操作：**
- 點擊「恢復」：取消忽略，恢復至未對應狀態

## 分析統計（Analytics）

`GET /analytics/`

### 年度熱力圖

GitHub 風格的每日列印活動熱力圖，顯示過去一年每天的列印次數。色塊越深表示任務越多。

點擊任一色塊可進入**每日報告頁**。

### 每日報告

`GET /analytics/day/<YYYY-MM-DD>`

點擊熱力圖色塊後進入，顯示該日的詳細資訊：

- **任務圖庫**：當日所有列印任務的封面圖、名稱、狀態
- **列印時間軸**：以時間軸方式呈現當日各任務的開始與結束時間
- **耗材使用摘要**：當日各材料的用量統計

### 材料分布

各材料（PLA、PETG、ABS 等）的用量佔比圓餅圖。

### 月度趨勢

歷史列印任務數量與總重量折線圖。支援**滑動平移**（拖曳圖表）查看更早的歷史資料，附重置按鈕可還原視角。

### 時長分布

列印時長的直方圖，顯示各時長區間的任務數量分布。

### 週間活動分布

以柱狀圖顯示一週七天各天的列印次數分布，可了解習慣列印的時間規律。

### 列印機使用率

各列印機的任務數、總重量、總列印時數比較圖。

### 成本分析

以圓餅圖依耗材成本計算各材料的花費，並附耗材成本排名。需在 Spool 中設定價格才能顯示正確數據。

## 設定（Settings）

`GET /settings/`

### Bambu Cloud 登入

**Step 1：帳密**
1. 輸入 Bambu Lab 帳號（Email）與密碼
2. 點擊「登入」

**Step 2：驗證碼 / 2FA**
- 若帳號啟用 2FA，輸入收到的驗證碼
- 點擊「驗證」完成登入

登入成功後 Token 自動存入資料庫，有效期約 3 個月。

### Token 狀態

顯示目前 Token 的前 8 字元（遮罩顯示），以及是否已設定 Token。

### 手動同步

點擊「立即同步」按鈕，從 Bambu Cloud 拉取最新列印歷史。

同步進行中頁面自動每 2 秒輪詢狀態（HTMX），顯示進度或完成/錯誤訊息。

### 自動同步

設定自動同步間隔（分鐘），0 = 關閉。

設定後重啟程式生效，或在設定頁直接更新排程。

### 時區設定

選擇 UTC 偏移值（如 UTC+8 對應台灣時間），設定後任務列表、任務詳情、分析統計、耗材詳情等頁面的時間戳記皆自動轉換為本地時間顯示。

> 備份與排程狀態的時間目前使用伺服器本地時間，不受此設定影響。

### 資料庫備份

**手動備份：** 點擊「立即備份」，備份儲存至 `data/backups/`。

**自動備份：** 設定備份間隔（分鐘）與保留份數（預設 7 份）。

**備份列表：** 顯示現有備份，可點擊「還原」從指定備份還原。

還原流程：
1. 點擊「還原」後，系統先自動備份**當前資料庫**（保留退路）
2. 出現確認提示，須手動輸入確認字詞
3. 確認後執行還原，覆蓋現有資料庫

> 注意：若自動備份失敗，還原操作會被中止以保護資料安全。
